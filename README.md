# Agent Conductor

Claude Code / Gemini CLI のマルチセッション管理デスクトップアプリ。非エンジニアでもターミナルを意識せず AI エージェントを使えることを目指して設計されています。

複数の AI エージェントセッションをタブで並列管理し、セッションの自動保存・復元、タスク管理、ファイルツリーを統合した Electron アプリケーションです。

> **前提**: このアプリは **Claude Code CLI** や **Gemini CLI** がインストール済みの環境で使うことを前提としています。アプリ自体にはAIエージェントは含まれておらず、ターミナル上でCLIを起動・管理するラッパーです。

## 機能一覧

### タブ管理
- **マルチタブ**: Claude / Gemini / 素のターミナルをタブで並列起動
- **タブ追加**: 右上の `+` ボタンから Claude / Gemini / Terminal を選択
- **タブ名の自動生成**: 最初の入力から末尾の動詞・助詞を除去して短いタイトルを自動生成
- **タブ名編集**: タブをダブルクリックで名前を変更
- **タブ並べ替え**: タブをドラッグで順序変更
- **タブ閉じる**: Claude/Gemini セッションがある場合は確認ダイアログ表示
- **閉じたタブの復元**: `↺` ボタンから最近閉じたタブを復元
- **Split view（左右2分割）**: タブを右クリック →「新しい分割ビューにタブを追加」、または Cmd+\ で分割トグル（Chromeのタブメニュー準拠）
  - 右クリックしたタブがアクティブタブの隣に表示される（アクティブタブ自身を右クリックした場合は隣のタブとペアに）
  - 分割中に第3のタブで「新しい分割ビューにタブを追加」すると、フォーカス中ペインの相方が差し替わる
  - 中央の divider をドラッグで幅調整（再起動後も保持）
  - 分割中のタブを右クリック →「分割ビューから削除」で外す（相方が単独表示に戻る）。右ペイン右上の × または Cmd+\ でも分割解除
  - クリックしたペインがフォーカスされ（アクセント色の枠で表示）、タブ切り替え・入力バー・ファイルツリーはフォーカス中のペインに作用

### セッション管理
- **自動保存**: アプリ終了時にタブの状態（セッションID、作業ディレクトリ、タブ名等）を自動保存
- **自動復元**: アプリ起動時に前回のセッションを `--resume` で復元
- **復元中表示**: 復元待ちのタブにはタブバー・サイドバーに "Resuming..." と表示
- **復元失敗時の自動フォールバック**: `--resume` が失敗した場合、自動的に新しいセッションを開始

### サイドバー
- **セッション一覧**: 全タブのステータスをリアルタイム表示
  - 緑パルスドット: エージェント実行中
  - グレードット: アイドル状態
  - "Thinking..." / ツール名: Claude/Gemini が実行中のアクションを表示
- **タスク管理**: サイドバー下部にタスクリスト
  - テキスト入力で追加
  - チェックで完了
  - ダブルクリックで編集
  - Claude の出力に `[[TASK: タスク名]]` があると自動追加
  - `[[TASK-SETALL: JSON]]` でタスクリスト一括更新
- **タスク送信**: タスク横の ◆ ボタンで新しいエージェントタブにタスクを送信

### ファイルツリー
- サイドバーのフォルダアイコンで表示/非表示を切り替え
- アクティブタブの作業ディレクトリに自動追従
- パス表示をクリックして手動でディレクトリを指定（固定モード）
- 拡張子ごとの色付きアイコン（TS/JS/JSON/CSS/PY/SH 等）
- ファイルをダブルクリック: ターミナルにパスを挿入
- ファイルを右クリック: エディタで開く / パスをコピー
- フォルダを右クリック: エディタで開く / パスをコピー / ルートに設定

### フローティング入力バー（Cmd+I）
- ターミナル外から長文・ファイルを Claude に送信できる入力ウィンドウ
- **テキスト入力**: Shift+Enter で改行、Enter または Cmd+Enter で送信（設定で切り替え）
- **ファイル添付**: 📎 ボタンまたはドラッグ＆ドロップでファイルをチップ表示
- **クリップボード画像**: ペーストで `/tmp/paste-xxx.png` として自動添付
- 高さはドラッグでリサイズ（設定は再起動後も保持）
- **送信モード**（設定から変更可能）:
  - `direct`: 送信後 Enter まで自動入力（ファイル添付時は画像UIの確認も自動化）
  - `manual`: テキストを貼り付けるだけで Enter は打たない

### ターミナル
- xterm.js ベースのフルターミナルエミュレータ
- テキストを選択して Backspace/Delete で一括削除
- macOS 標準キーバインド（⌘+C / ⌘+V 等）がそのまま使える（通常のターミナルアプリでは Ctrl ベースが必要な場合があるが、Electron アプリのため OS 標準のキー操作が自然に動作する）
- フォントサイズは設定から変更可能

### 設定（ステータスバーの ⚙ から）
- **テーマ**: Dark / Light
- **フォントサイズ**: 11〜20px（スライダー）
- **言語**: EN / JA（ヘルプも連動）
- **アクセントカラー**: 7色プリセット + カスタム Hex コード入力
  - フッターバー、UI のアクセント要素に即時反映
- **エディタ**: ファイルツリーからファイルを開く際のエディタコマンド
  - プリセット: macOS Default / code / cursor / vim / nvim / subl
- **入力バー送信モード**: `Enter` / `Cmd+Enter` / `ボタン`
- **自動送信モード**: `direct`（Enter自動）/ `manual`（貼り付けのみ）

### カラム幅調整
- サイドバー / ファイルツリー / ターミナルのカラム幅をドラッグで調整
- 幅は再起動後も保持

## Tips: Claude Code スキルとの連携

Agent Conductor のサイドバータスク機能は、Claude Code のカスタムスキル（スラッシュコマンド）と組み合わせると強力に機能します。

### `/add-task` — タスクの自動登録

Claude Code のセッション内で `/add-task` を実行すると、`[[TASK: ...]]` 形式の出力が生成され、サイドバーにタスクが自動追加されます。

- `/add-task READMEを更新する` → 指定したタスクを即座に登録
- `/add-task`（引数なし） → 直前の会話から残タスクを自動洗い出して一括登録

### `[[SEND: タブ名 :: 本文]]` — タブ間 Agent 会話

Agent の出力に `[[SEND: タブ名 :: 本文]]` が含まれると、メインプロセスが宛先タブの Agent にメッセージを自動配送します。Agent 同士をタブをまたいで会話させる仕組みです。

使い方:

1. 各タブに名前を付ける（タブをダブルクリックして命名）
2. Agent にプロトコルを教える。各 Agent の**グローバル指示ファイル**に以下のサンプルを追記する（指示ファイル名は Agent ごとに異なる）:
   - Claude Code → `~/.claude/CLAUDE.md`
   - Codex CLI → `~/.codex/AGENTS.md`（プロジェクトの `AGENTS.md` でも可）
   - Gemini CLI → `~/.gemini/GEMINI.md`

   ```markdown
   ## Agent Conductor タブ間連携

   環境変数 `AGENT_CONDUCTOR=1` が設定されている場合、あなたは Agent Conductor のタブ内で動作している。
   他のタブの Agent にメッセージを送るときは、出力に次の記法を含めること:

   [[SEND: <宛先タブ名> :: <本文>]]

   - 本文はユーザーからの依頼文をそのまま転送するのではなく、宛先の Agent がそのまま着手できる内容を自分で組み立てる
   - 宛先タブにはあらかじめ名前が付いている必要がある。宛先タブ名が分からないときは送る前にユーザーに確認する
   - 送信したメッセージは宛先タブに `[from: 送信元タブ名] 本文` の形式で届く
   ```
3. Agent が `[[SEND: レビュー担当 :: 実装が終わったのでレビューして]]` のように出力すると、「レビュー担当」タブに `[from: 送信元タブ名] 本文` の形式で届き、自動で Enter が送信される

動作仕様:

- **宛先解決**: タブ名の完全一致 → 前方一致の順で照合（曖昧・不一致の場合は配送せずログに記録）
- **busy 時のキューイング**: 宛先の Agent が出力中（直近3秒以内に出力あり）の場合はキューに積み、アイドルになってから配送
- **二重送信ガード**: 一度配送したメッセージはタブが生きている間ずっと重複検出される（TUI の再描画・リサイズによる再出現やセッション復元時のリプレイでも二重配送されない。折り返し位置の違いは空白正規化で同一視）。ユーザーが入力欄から直接打った `send:タブ名 本文` は対象外で、同一文面でも何度でも送れる

### `/handover` — セッション引き継ぎ

コンテキストが長くなったり、作業を区切りたいときに `/handover` を実行すると：

1. 現在のセッションから引き継ぎテキスト（完了事項・残タスク・決定事項）を自動生成
2. 引き継ぎファイルを保存し、クリップボードにコピー
3. サイドバーに「引き継ぎ: タイトル → ファイルパス」のタスクを自動追加
4. タスク横の ◆ ボタンを押すと、新しい Claude タブが開き、引き継ぎ内容が自動で送信される

**引き継ぎはサイドバーにタスクとして残るので、すぐに新セッションを立ち上げる必要はありません。** 好きなタイミングで ◆ ボタンを押せば、コンテキストを引き継いだ新セッションが開始されます。

## インストール

### DMG から
1. DMG を開いて Applications フォルダにドラッグ
2. ターミナルで以下を実行して Gatekeeper を解除：
   ```bash
   xattr -cr /Applications/Agent\ Conductor.app
   ```
3. 以降はダブルクリックで起動できる

> **なぜこの手順が必要？** macOS の Gatekeeper は App Store 外・未公証のアプリをブロックします。xattr コマンドで拡張属性を削除することで回避できます。**アップデートのたびに必要です**（新しい DMG をインストールするたびに実行してください）。

### 開発環境
```bash
cd agent-conductor
npm install
npm run dev
```

### ビルド
```bash
npm run dist
# → dist/Agent Conductor-x.x.x-arm64.dmg
```

